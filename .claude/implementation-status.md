# Статус реализации сайта

Решили не ждать разблокировки Figma MCP и перешли сразу к коду (Vite, см. `tech-stack.md`) — макет из Figma (секции 1–7, см. `figma-mockup-progress.md`) использован как референс визуального языка, дальше итерации идут прямо в браузере.

## Что сделано

Полностью собран и запускается одностраничник по `content-structure.md` — все 13 секций:

1. Обложка (монограмма «А & Ю», имена, дата, CSS-анимация «открытия конверта»)
2. Приветственный текст
3. Таймер обратного отсчёта (реальный JS-таймер до 17.10.2026 15:00, не демо-заглушка)
4. Дата, время и место (роспись/welcome-зона/банкет)
5. Программа дня (полный тайминг, помечен как предварительный)
6. Локация и карта (плейсхолдер под Яндекс.Карты)
7. Дресс-код (чёрно-белые свотчи)
8. Love story (таймлайн, Lorem-текст, фото-плейсхолдеры)
9. RSVP-форма (все поля из брифа, клиентская валидация, honeypot-защита от спама)
10. Анкета для гостей
11. Второй день свадьбы
12. Контакты организаторов
13. Футер

**Технически:**
- Vite + vanilla JS-модули (`src/sections/*.js`), CSS custom properties как дизайн-токены (`src/styles/tokens.css`) — 1:1 с переменными, заведёнными в Figma
- Шрифты Yeseva One / Montserrat / Marck Script самохостятся через `@fontsource/*`
- Таймер, scroll-reveal анимации (IntersectionObserver), обработка форм — протестированы через Playwright (клик, заполнение, валидация, submit) — работает без ошибок в консоли
- RSVP/анкета готовы отправлять данные в Google Apps Script Web App — сейчас работают в demo-режиме (логируют в консоль), т.к. `APPS_SCRIPT_URL` в `src/sections/formHandler.js` ещё не настроен (нужен реальный Apps Script после того, как заказчик подтвердит Google-таблицу)
- Проверена адаптивность: мобильный (390px) и desktop (1440px) — love story переключается на зигзаг-раскладку, блок даты/времени — на строчную

## Как запустить

```bash
npm install
npm run dev       # dev-сервер с HMR
npm run build     # прод-сборка в dist/
```
