# Сайт-приглашение на свадьбу — схема блоков

Обзор рынка русскоязычных конструкторов свадебных сайтов и типовой структуры одностраничника-приглашения.

## 1. Схема основных блоков (must have)

Порядок соответствует типичному сценарию прокрутки страницы гостем.

1. **Обложка / Конверт**
   Имена молодожёнов, дата свадьбы, декоративная анимация «открытия конверта» при заходе на сайт.
2. **Таймер обратного отсчёта**
   Дни/часы/минуты до события — создаёт ощущение предвкушения.
3. **Дата, время и место**
   Ключевые факты по загсу/церемонии и банкету.
4. **Программа дня (тайминг)**
   Расписание по часам: сбор гостей, церемония, фотосессия, банкет, конец вечера.
5. **Локация и карта проезда**
   Интерактивная карта (Яндекс/Google Maps), адрес, как добраться (машина/такси/парковка).
6. **Дресс-код**
   Палитра цветов торжества, рекомендации по стилю одежды.
7. **RSVP-форма**
   Подтверждение присутствия, количество гостей, детские места, аллергии/пожелания по меню.
8. **Контакты организаторов**
   Телефон/Telegram координатора на случай вопросов у гостя.
9. **Футер**
   Финальное благодарственное слово, дата, имена, иконки соцсетей (по желанию).

## 2. Дополнительные блоки (опционально, по желанию пары)

- **Love story / история пары** — таймлайн знакомства, предложения, помолвки с фото.
- **Фотогалерея** — совместные фото пары; отдельный раздел может открываться уже после свадьбы для загрузки фото от гостей.
- **Список подарков / вишлист** — ссылки на маркетплейсы или номер счёта «в конверт».
- **Мини-анкета/опрос для гостей** — пожелания по музыке, конкурсам, шуточные вопросы.
- **Персональное обращение к гостю** — при переходе по именной ссылке сайт обращается к гостю по имени.
- **Информация о свадебном застолье/меню** — варианты блюд, отметка для RSVP.
- **Размещение и трансфер** — рекомендации отелей рядом с площадкой, организованный автобус.
- **Хэштег свадьбы** — для соцсетей.
- **Фоновая музыка** — трек, включающийся при открытии сайта.
- **FAQ** — ответы на частые вопросы (можно ли с детьми, где переодеться и т.д.).
- **Сохранение в PDF / добавление в календарь** — кнопка «добавить в Google/Apple Calendar».
- **QR-код приглашения** — для печатных версий, ведущих на сайт.
- **Обратный отсчёт до других событий** — девичник/мальчишник, роспись отдельно от банкета.
- **Многоязычность** — для гостей-иностранцев.
- **Второй день свадьбы** — отдельный блок программы, если празднование продолжается.

## 4. Шрифты Google Fonts

Сайт русскоязычный, поэтому все шрифты в списке проверены на **поддержку кириллицы**. Обычно для свадебного стиля берут пару: декоративный шрифт для заголовков/имён + нейтральный для основного текста.

### Каллиграфические / скрипт (заголовки, имена, монограмма)
- [Marck Script](https://fonts.google.com/specimen/Marck+Script) — рукописный, изящный, хорошо смотрится на именах молодожёнов
- [Bad Script](https://fonts.google.com/specimen/Bad+Script) — живой почерк, менее формальный
- [Playball](https://fonts.google.com/specimen/Playball) — округлый скрипт, дружелюбный
- [Comforter](https://fonts.google.com/specimen/Comforter) — декоративный каллиграфический, для акцентов (не для длинного текста)

### Декоративные serif (даты, заголовки блоков)
- [Yeseva One](https://fonts.google.com/specimen/Yeseva+One) — контрастный элегантный заголовочный шрифт
- [Playfair Display](https://fonts.google.com/specimen/Playfair+Display) — классика для wedding-дизайна
- [Cormorant Garamond](https://fonts.google.com/specimen/Cormorant+Garamond) — тонкий, «дорогой» вид
- [Forum](https://fonts.google.com/specimen/Forum) — античный, торжественный

### Нейтральный serif/sans (основной текст, программа дня, RSVP)
- [PT Serif](https://fonts.google.com/specimen/PT+Serif) — российская типографика (ParaType), отлично читается
- [Philosopher](https://fonts.google.com/specimen/Philosopher) — элегантный, но универсальный
- [Montserrat](https://fonts.google.com/specimen/Montserrat) — геометричный sans, хорошо для UI-элементов (кнопки, формы)
- [Golos Text](https://fonts.google.com/specimen/Golos+Text) — современный российский sans, отличная читаемость на мобильных

### Рекомендуемые пары для старта
1. **Marck Script** (имена/заголовок) + **PT Serif** (основной текст)
2. **Yeseva One** (заголовки блоков) + **Montserrat** (текст, кнопки, формы)
3. **Cormorant Garamond** (даты, цитаты) + **Golos Text** (UI и основной текст)

## 5. Какая информация нужна, чтобы приступить к макету

Чек-лист данных от заказчика перед стартом дизайна:

**Событие**
- [ ] Имена молодожёнов (в каком виде писать — полные/сокращённые, порядок) - Александр, Юлия
- [ ] Дата и время церемонии/росписи - роспись 15:00 welcome зона 18:00
- [ ] Дата и время банкета (если отличается)
- [ ] Адреса площадок (загс, банкет, при необходимости — сбор гостей) + ссылки на карты Добавим позже
- [ ] Подробная программа дня по часам - Заполним позже, используй стандартный вариант

**Стиль**
- [ ] Референсы/примеры сайтов или пригласительных, которые нравятся (можно из раздела «Примеры» выше) Выбери сам
- [ ] Цветовая палитра / дресс-код торжества Черный
- [ ] Есть ли готовый логотип, монограмма или фирменный стиль свадьбы - пока ничего нет, создай сам
- [ ] Фотографии пары для обложки, галереи и love story (или дата фотосессии, если фото ещё нет) Используй заглушки

**Контент**
- [ ] Текст истории знакомства (love story), если блок нужен используй текст Lorem
- [ ] Список опциональных блоков из раздела 2, которые нужно включить История пары, анкета для гостей(вопросы составим позже), блок о втором дне
- [ ] Контакты организатора/координатора для гостей - используй заглушку, потом добавим
- [ ] Ссылки на вишлист / реквизиты для подарков, если применимо - не нужно
- [ ] Текст приветствия/обращения к гостям - Придумай сам

**RSVP и функциональность**
- [ ] Нужна ли форма подтверждения присутствия и какие поля собирать (кол-во гостей, дети, аллергии/меню) - Да, определим потом
- [ ] Куда должны приходить ответы гостей (email, Telegram-бот, Google-таблица) Google таблица
- [ ] Нужны ли именные ссылки/персональные обращения к гостям нет
- [ ] Ограничение по устройствам — только мобильная версия или desktop тоже важен. Полный адаптив

**Технические детали**
- [ ] Домен (уже есть/нужно подобрать) и срок жизни сайта (до/после свадьбы) Подбери варианты
- [ ] Нужна ли многоязычность - нет
- [ ] Планируемая дата запуска сайта (с каким запасом до рассылки приглашений) - есть несколько месяцев.
